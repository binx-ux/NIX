Introduction

Webhookr is a browser-based tool for constructing and sending Discord embeds via webhooks. No installation, no account, no dependencies.

Use the Builder tab to compose an embed visually. The live preview shows you exactly how Discord will render it. When ready, paste your webhook URL and click Send.

  • All processing happens in the browser
  • Webhook URL is never stored to any server
  • Send history persists for the current session only

Finding Your Webhook URL

Webhooks are created per channel in Discord's server settings.

  • Open your Discord server and go to Server Settings
  • Navigate to Integrations then Webhooks
  • Click New Webhook, choose a channel, and copy the URL
  • The URL format is: https://discord.com/api/webhooks/ID/TOKEN

Keep your webhook URL private. Anyone with the URL can send messages to your channel.

Embed Structure

An embed is a rich content card sent alongside or in place of plain message content. The JSON structure sent to Discord looks like this:

{ "username": "My Bot", "content": "Optional outer text", "embeds": [ { "title": "Embed Title", "description": "Embed body text", "color": 5793266, "author": { "name": "Author Name", "url": "https://...", "icon_url": "https://..." }, "fields": [ { "name": "Field Name", "value": "Field Value", "inline": true } ], "image": { "url": "https://..." }, "thumbnail": { "url": "https://..." }, "footer": { "text": "Footer text", "icon_url": "https://..." }, "timestamp": "2025-01-01T00:00:00.000Z" } ] }

The color field is a decimal integer. Webhookr converts hex color input to decimal automatically.

Fields

Fields are key-value pairs rendered inside the embed. Each field has a name, a value, and an optional inline boolean.

PropertyTypeDescription
namestringField label. Required. Max 256 characters.
valuestringField body. Required. Max 1024 characters.
inlinebooleanWhen true, Discord tries to place the field next to adjacent inline fields (up to 3 per row).

Raw API Usage

You can send embeds directly from any HTTP client. The endpoint accepts a POST request with a JSON body.

curl -X POST \ -H "Content-Type: application/json" \ -d '{"embeds":[{"title":"Hello","description":"World","color":5793266}]}' \ https://discord.com/api/webhooks/ID/TOKEN

Rate limit: Discord allows approximately 30 requests per 60 seconds per webhook. A 429 response means you are being rate limited. Check the retry_after field in the response body.
